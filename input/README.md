# 📥 Папка для исходных материалов

> **Эта папка — ваша "сырая" база знаний для ВКР.**
> Положите сюда всё, что уже наработали — AI-ассистент будет использовать
> эти материалы для оформления диплома.

---

## 🛡️ ГАРАНТИЯ СОХРАННОСТИ

**Файлы в этой папке НИКОГДА не будут изменены AI-ассистентом.**

AI имеет право только:
- ✅ Читать ваши файлы
- ✅ Копировать содержимое в главы ВКР
- ✅ Использовать ваши формулировки

AI НЕ будет:
- ❌ Редактировать ваши файлы
- ❌ Удалять ваши файлы
- ❌ Переименовывать ваши файлы

---

## 📁 Структура папок

### `drafts/` — Черновики текстов

Положите сюда:
- Наброски глав (Markdown, TXT, любой текстовый формат)
- Конспекты, заметки
- Готовые куски текста

**Примеры имён файлов:**
```
drafts/
├── введение.md
├── глава1-исследование.md
├── глава2-проектирование.txt
├── глава3-реализация.md
├── заключение.md
├── аннотация-рус.md
├── аннотация-eng.md
└── заметки-разные.txt
```

### `code/` — Исходный код

Положите сюда:
- Код вашего проекта/системы
- Скрипты, конфигурации
- Примеры использования

Этот код будет оформлен как листинги в приложениях.

```
code/
├── main.py
├── utils/
│   ├── helpers.py
│   └── config.py
├── sql/
│   └── schema.sql
└── docker-compose.yml
```

### `data/` — Данные и результаты

Положите сюда:
- Результаты экспериментов
- Таблицы с данными (CSV, JSON)
- Скриншоты интерфейса
- Диаграммы, схемы

```
data/
├── results.csv
├── metrics.json
├── screenshots/
│   ├── main-screen.png
│   └── dashboard.png
└── diagrams/
    ├── architecture.png
    └── sequence.png
```

### `references/` — Источники и литература

Положите сюда:
- Список литературы (в любом формате)
- Заметки по источникам
- Ссылки на статьи, книги
- PDF статей (опционально)

```
references/
├── sources.md          # Ваш список источников
├── notes-by-topic.md   # Заметки по темам
└── links.txt           # Полезные ссылки
```

## 💡 Советы по подготовке материалов

### Для текстов

1. **Не беспокойтесь о форматировании** — AI переведёт в Typst
2. **Пишите как удобно** — Markdown, TXT, даже без разметки
3. **Используйте понятные имена файлов** — `глава1.md` лучше чем `doc1.md`

### Для кода

1. **Рабочий код** — который вы реально использовали
2. **С комментариями** — AI сохранит их в листингах
3. **Разумный размер** — очень длинные файлы разбейте на части

### Для изображений

1. **Достаточное разрешение** — минимум 150 DPI для печати
2. **Понятные имена** — `архитектура-системы.png` лучше чем `img1.png`
3. **PNG для схем**, JPG для фото/скриншотов

## 🚀 Что дальше?

1. Положите ваши материалы в соответствующие папки
2. Запустите AI-ассистент (Cursor, Claude, и т.д.)
3. Попросите: "Изучи материалы в input/ и помоги оформить ВКР"

AI прочитает ваши материалы и поможет:
- Распределить по главам
- Оформить по ГОСТ
- Создать листинги кода
- Сформировать список литературы
